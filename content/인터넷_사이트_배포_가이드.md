## 인터넷에서 접속 가능한 사이트로 만드는 방법

### 1. GitHub Pages

- 장점: 무료, 정적 사이트에 적합, 파일 이력 관리 가능
- 단점: 기본적으로 공개 저장소를 쓰면 내용이 인터넷에 공개됨
- 필요한 것: GitHub 계정, repository 생성, site 폴더 업로드 또는 GitHub Actions 설정

### 2. Cloudflare Pages / Netlify / Vercel

- 장점: 정적 사이트 배포가 쉽고 HTTPS 제공
- 단점: 계정 연결과 배포 설정 필요
- 필요한 것: 각 서비스 계정, GitHub 연동 또는 수동 업로드

### 3. 개인 PC 직접 공개

- 장점: 외부 서비스 없이 가능
- 단점: 보안, 방화벽, 공유기 포트포워딩, DDNS 관리가 필요하므로 권장하지 않음

## 공개 전 확인 사항

- Trend 문서에 개인 정보, 로컬 경로, API 키, 비공개 메모가 포함되어 있지 않은지 확인
- 인터넷 공개 범위를 public으로 할지 private 접근으로 제한할지 결정
- 자동 업데이트 결과가 공개되어도 괜찮은지 확인

## 운영 방식

- Trend 일일 업데이트 스크립트가 끝나면 site/content와 docs.json을 자동 갱신한다.
- 인터넷 배포를 붙이면 갱신된 site 폴더를 호스팅 서비스로 업로드하거나 git push하면 된다.
